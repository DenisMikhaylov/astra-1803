
### Лабораторная работа  «Сброс пароля root в Linux»

**Цель работы:** Научиться восстанавливать доступ к системе Linux путем сброса пароля root через различные методы загрузки, понимать механизмы защиты и уметь применять их в аварийных ситуациях.

**Теоретическая справка:**

**Что такое root?**
- root — это суперпользователь в Linux с полными правами доступа
- Пароль root — критический элемент безопасности системы
- Потеря пароля root не означает потерю системы

**Основные методы сброса пароля:**

1. **GRUB + recovery mode** — самый распространенный способ
2. **Загрузка с Live CD/USB** — универсальный метод
3. **Однопользовательский режим (Single User Mode)** — для систем без GRUB
4. **Использование init=/bin/bash** — для систем с systemd
5. **Chroot из Live-системы** — для полного восстановления

**Важно:** Все методы требуют физического доступа к серверу или доступа через консоль (iDRAC, IPMI, etc.)

# Сброс пароля root через GRUB Recovery Mode

## Подготовка

Для выполнения этой операции необходим:
- Физический доступ к серверу или доступ к консоли (iDRAC, IPMI, etc.)
- Клавиатура для ввода команд
- Время на перезагрузку системы

## Пошаговая инструкция

### Шаг 1: Перезагрузка системы и вход в GRUB

1. Перезагрузите систему:
```bash
sudo reboot
```
или нажмите кнопку перезагрузки на сервере.

2. Во время загрузки нажмите и удерживайте клавишу **Shift** (для BIOS) или **Esc** (для UEFI) для входа в меню GRUB.

3. Вы увидите меню загрузки с доступными ядрами.

### Шаг 2: Редактирование параметров загрузки

1. Выберите строку с нужным ядром (обычно первая).

2. Нажмите клавишу **e** для редактирования параметров загрузки.

3. Найдите строку, начинающуюся с `linux` или `linux16` (она содержит параметры ядра).

4. В конце этой строки добавьте один из следующих параметров:

**Для систем с systemd:**
```
systemd.unit=rescue.target
```

**Или для систем с init (SysVinit):**
```
single
```

**Или (альтернативный способ):**
```
init=/bin/bash
```

### Шаг 3: Загрузка в восстановительный режим

1. Нажмите **Ctrl+X** или **F10** для загрузки с новыми параметрами.

2. Система загрузится в recovery mode.

### Шаг 4: Монтирование файловой системы

**Важно:** Корневая файловая система обычно монтируется в режиме **только для чтения (ro)**.

1. Проверьте текущее состояние монтирования:
```bash
mount | grep " / "
```

2. Перемонтируйте корневую файловую систему в режим **чтения-записи (rw)**:

**Для systemd:**
```bash
mount -o remount,rw /
```

**Или с использованием mount:**
```bash
mount -o remount,rw /dev/sda1 /
```

> **Примечание:** Уточните устройство корневой ФС (может быть `/dev/sda1`, `/dev/mapper/root` и т.д.)

### Шаг 5: Сброс пароля root

1. Сбросьте пароль root:
```bash
passwd root
```

2. Введите новый пароль дважды.

3. Если нужно создать нового пользователя с правами root:
```bash
useradd -m -G wheel admin
passwd admin
usermod -aG wheel admin  # Для систем с sudo
```

### Шаг 6: Выход и перезагрузка

1. Выйдите из оболочки:
```bash
exit
```

2. Перезагрузите систему:
```bash
reboot
```
или
```bash
exec /sbin/init
```

3. Проверьте новый пароль:
```bash
su -
# или
ssh root@localhost
```

## Альтернативные методы

### Метод через init=/bin/bash

1. В GRUB добавьте параметр `init=/bin/bash`.

2. При загрузке выполните:
```bash
mount -o remount,rw /
passwd root
mount -o remount,ro /
exec /sbin/init
```
