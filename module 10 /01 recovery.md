### Способ 1: Восстановление с установочного носителя (для версии 1.8)

Это основной и рекомендуемый способ для Astra Linux Special Edition 1.8.

1.  **Загрузка**: Загрузитесь с установочного носителя Astra Linux Special Edition 1.8.
2.  **Доступ к консоли**: Дождитесь приглашения программы установки, затем нажмите клавишу `<F10>` и в появившемся меню выберите пункт **«Выйти в консоль»**. Вы перейдете в командную строку.
3.  **Монтирование системы**: Определите имя вашего корневого раздела (например, с помощью `lsblk`) и примонтируйте его:
    ```bash
    mount /dev/sdX2 /mnt
    ```
    (замените `/dev/sdX2` на ваш корневой раздел).
4.  **Подготовка окружения** (опционально): Если вам нужно выполнить сложные операции (например, установку пакетов), примонтируйте специальные каталоги:
    ```bash
    mount --bind /dev/pts /mnt/dev/pts
    mount -t sysfs /sys /mnt/sys
    mount -t proc /proc /mnt/proc
    ```
5.  **Переход в систему**: Выполните команду `chroot`, чтобы войти в вашу основную систему и получить к ней полный доступ:
    ```bash
    chroot /mnt
    ```
    После этого вы можете выполнять любые команды по восстановлению.

### Способ 2: Использование загрузчика GRUB2

Этот способ работает, если меню загрузчика GRUB2 включено. Однако здесь есть важный нюанс.

1.  **Режим Recovery**: Выберите пункт **"recovery mode"** в меню GRUB.
2.  **Проблема с паролем root**: В стандартной конфигурации Astra Linux 1.8 пользователь `root` заблокирован. При попытке загрузиться в режиме recovery вы увидите сообщение: `Cannot access to the console, the root account is locked`.
3.  **Обход блокировки**: Чтобы обойти это и получить доступ к оболочке с правами root:
    1.  В меню загрузчика нажмите клавишу **`e`** для редактирования параметров загрузки.
    2.  Найдите строку, начинающуюся со слова `linux`.
    3.  Добавьте в конец этой строки параметр ядра **`init=/bin/bash`**.
    4.  Нажмите `Ctrl+X` или `F10` для загрузки.
4.  **Результат**: Система загрузится в командную оболочку `/bin/bash` с правами суперпользователя, что позволит вам выполнить необходимые действия по восстановлению.

### Что делать, если `initrd` поврежден?

Если проблема связана с повреждением файлов `initrd` в каталоге `/boot`, вы можете попытаться восстановить их. Для этого в среде восстановления (способ 1) выполните следующие шаги:

1.  **Удаление старых/поврежденных образов** (по желанию):
    ```bash
    rm /boot/*-<старая-версия>-hardened
    ```
2.  **Пересоздание initrd**:
    ```bash
    update-initramfs -u -k <версия-ядра>
    ```
    Если нужно создать для всех доступных ядер:
    ```bash
    update-initramfs -u
    ```

### Важные моменты для работы в консоли восстановления

*   **Отсутствие сети**: В стандартном режиме восстановления сетевые подключения не работают.
*   **Рекомендация по пакету `ncurses-term`**: Для более комфортной работы в "синей консоли" режима восстановления рекомендуется заранее установить пакет `ncurses-term`. Это даст возможность использовать полноценные псевдографические утилиты (например, `mc`). Если пакет не установлен, можно переключиться в "черную консоль" комбинацией `Ctrl+Alt+F2`.

В целом, подход к восстановлению в Astra Linux Special Edition 1.8 отличается от более ранних версий и ориентирован на использование возможностей установочного носителя, что делает процесс работы с `initrd` и ядром более управляемым.
