### Способ 1: Восстановление с установочного носителя (рекомендуемый)

Это основной и надёжный способ, так как установочный носитель содержит всё необходимое для работы с дисками и файловыми системами, включая LVM .

#### Шаг 1. Загрузка в режим восстановления
1.  Загрузитесь с установочного носителя Astra Linux Special Edition 1.8.
2.  В процессе загрузки дождитесь появления приглашения программы установки.
3.  Нажмите клавишу **`<C>`**, чтобы попасть в командную строку установщика .
4.  После запуска программы установки нажмите клавишу **`<F10>`**.
5.  В открывшемся меню выберите пункт **«Выйти в консоль»** . Вы окажетесь в командной строке оболочки.

#### Шаг 2. Активация LVM и монтирование корневой файловой системы
Так как ваша система использует LVM, перед монтированием необходимо активировать логические тома .

1.  **Активируйте все тома LVM**:
    ```bash
    lvm vgchange -a y
    ```
    Эта команда сделает доступными все логические тома в системе .

2.  **Определите имена ваших логических томов**:
    ```bash
    lvm lvs
    ```
    Вывод команды покажет список томов. Обычно корневой том называется `lv_root` или имеет похожее имя в группе томов (например, `VG/lv_root`) .

3.  **Создайте точку монтирования и примонтируйте корневой том**:
    ```bash
    mount /dev/<имя-группы-томов>/<имя-корневого-тома> /mnt
    ```
    Например, если группа томов называется `astra`, а корневой том — `root`, команда будет выглядеть так:
    ```bash
    mount /dev/astra/root /mnt
    ```
    Если у вас также отдельный том для `/home`, его можно примонтировать для полноты доступа :
    ```bash
    mount /dev/<имя-группы-томов>/<имя-тома-home> /mnt/home
    ```

4.  **Подготовьте окружение для перехода в систему** (это необходимо для корректной работы команд восстановления) :
    ```bash
    mount --bind /dev /mnt/dev
    mount --bind /dev/pts /mnt/dev/pts
    mount --bind /sys /mnt/sys
    mount --bind /proc /mnt/proc
    ```

#### Шаг 3. Вход в систему и восстановление initrd
Теперь вы переключитесь в вашу основную систему, чтобы выполнить команды по пересозданию `initrd` .

1.  **Выполните смену корневого каталога**:
    ```bash
    chroot /mnt
    ```

2.  **Определите версию ядра**, для которой нужно восстановить `initrd`. Посмотрите содержимое каталога `/boot`:
    ```bash
    ls -la /boot
    ```
    Найдите файлы ядра (обычно имеют вид `vmlinuz-<версия>-hardened`). Запомните версию.

3.  **Пересоздайте `initrd` для нужной версии ядра**:
    ```bash
    update-initramfs -u -k <версия-ядра>
    ```
    Например:
    ```bash
    update-initramfs -u -k 5.4.0-162-generic
    ```
    Если вы хотите обновить `initrd` для всех установленных ядер, выполните:
    ```bash
    update-initramfs -u
    ```

#### Шаг 4. Завершение работы и перезагрузка
1.  **Выйдите из сессии `chroot`**:
    ```bash
    exit
    ```

2.  **Отмонтируйте все временно примонтированные каталоги** :
    ```bash
    umount -R -l /mnt
    ```

3.  **Извлеките установочный носитель**.
4.  **Перезагрузите компьютер**:
    ```bash
    reboot
    ```
