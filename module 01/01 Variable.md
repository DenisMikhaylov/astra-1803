
### Лабораторная работа №1: «Переменные и подстановки в Bash»

**Цель работы:** Научиться объявлять, изменять, подставлять и преобразовывать переменные различных типов, используя механизмы синтаксического анализа Bash без применения управляющих конструкций.

**Теоретическая справка:**
В Bash все переменные по умолчанию строковые. Однако существуют **арифметические подстановки** `$(( ... ))` для чисел и **ассоциативные массивы** (ключ-значение). В работе мы используем:
1. **Скалярные строковые** переменные.
2. **Целочисленные** переменные (с `declare -i` или арифметикой).
3. **Индексированные массивы** (нумерация с 0).
4. **Ассоциативные массивы** (требуют `declare -A`).

### Варианты заданий (Выберите один по списку группы)

#### Вариант 1 (Системный администратор)
Напишите скрипт, который:
1. Принимает на вход **путь к директории** и **расширение файла** (например, `.txt`).
2. Создает строковую переменную `RESULT`, в которую записывает команду `ls -l`.
3. Создает целочисленную переменную `COUNT`, равную количеству файлов с данным расширением в указанной директории (используйте подстановку `$(...)`).
4. Создает массив `FIRST_FIVE`, который содержит первые 5 имен файлов из вывода `ls`.
5. Создает переменную `REPORT`, которая содержит строку: *"В папке [путь] найдено [COUNT] файлов с расширением [расширение]. Первые два: [элемент0], [элемент1]"*.
6. Выводит `REPORT` на экран.

#### Вариант 2 (Разработчик)
Напишите скрипт, который:
1. Объявляет **ассоциативный массив** `CONFIG` с ключами: `user`, `home`, `shell`, `group`.
2. Заполняет эти ключи значениями из текущей системы: `USER`, `HOME`, `SHELL`, `GROUPS` (первый элемент из массива групп).
3. Создает целочисленную переменную `UID_NUM` и записывает в нее числовое значение `UID` текущего пользователя, увеличенное на 100.
4. Создает строковую переменную `SUMMARY`, которая содержит многострочный текст с использованием `\n` (или переносов кавычками) со всеми значениями из `CONFIG` и `UID_NUM`.
5. Выводит `SUMMARY` и завершает работу с кодом возврата, равным числу символов в переменной `SUMMARY` (используйте `exit ${#SUMMARY}`).

#### Вариант 3 (Аналитик данных)
Напишите скрипт, который:
1. Принимает строку с произвольным текстом через `read -p`.
2. Сохраняет её в переменную `INPUT_TEXT`.
3. Создает переменные:
   - `LEN` = длина текста.
   - `FIRST` = первый символ текста.
   - `LAST` = последний символ текста.
   - `CLEAN` = текст, преобразованный в нижний регистр.
4. Создает **индексированный массив** `WORDS`, который получается разбиением `INPUT_TEXT` по пробелам (используйте `()` и подстановку).
5. Создает строку `STATS`, которая содержит количество слов (`${#WORDS[@]}`), первый и последний символы.
6. Все вычисления выполняются **без использования внешних утилит** (например, `wc`, `tr`, `sed` не использовать, только встроенные средства Bash). Вывод `STATS`.

#### Вариант 4 (DevOps)
Напишите скрипт, который:
1. Объявляет **ассоциативный массив** `SERVICES` с ключами: `web`, `db`, `cache`.
2. Присваивает им значения: `web=nginx`, `db=postgresql`, `cache=redis`.
3. Создает **индексированный массив** `RUNNING`, который содержит значения всех трех ключей массива `SERVICES` (без ключей).
4. Создает целочисленную переменную `COUNTER` = количеству элементов в `RUNNING`.
5. Создает переменную `DEPLOY_CMD` = строку `"systemctl restart "`, после чего добавляет к ней через пробел все элементы массива `RUNNING` (используйте `"${RUNNING[*]}"`).
6. Создает переменную `BACKUP_CMD`, которая получается заменой в `DEPLOY_CMD` слова `restart` на `status`.
7. Выводит на экран `DEPLOY_CMD` и `BACKUP_CMD` в разных строках.

#### Вариант 5 (Сетевой инженер)
Напишите скрипт, который:
1. Принимает **IP-адрес** (например, `192.168.1.1`) и **порт** (например, `8080`) как аргументы командной строки (`$1`, `$2`).
2. Создает переменную `PROTOCOL` со значением `"http"`.
3. Создает целочисленную переменную `PORT_CHECK`, в которую записывается порт, но если порт не передан (`$2` пустой), то присвоить `80`.
4. Создает массив `URL_PARTS`, состоящий из: `PROTOCOL`, `$1`, `PORT_CHECK`.
5. Создает строковую переменную `FULL_URL` = `"$PROTOCOL://$1:$PORT_CHECK"`.
6. Создает переменную `PING_CMD` = `"ping -c 1 "` + первый октет IP-адреса (извлеките через подстановку `${var%%.*}` или другие манипуляции).
7. Выводит `FULL_URL` и `PING_CMD`.
