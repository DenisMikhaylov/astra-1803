Лабораторная работа №4: «Условные операторы и проверки в Bash»
Цель работы: Научиться использовать условный оператор if для ветвления логики скрипта, применять различные проверки (файлов, строк, чисел) и комбинировать их с переменными.

Теоретическая справка:

В Bash условия проверяются с помощью команды [ (синоним test). Основные типы проверок:

Файловые проверки:

-f — существует ли обычный файл

-d — существует ли директория

-e — существует ли объект (файл/директория)

-r — доступен для чтения

-w — доступен для записи

-x — доступен для выполнения

-s — файл не пустой

Строковые проверки:

-z — строка пуста (zero length)

-n — строка не пуста (non-zero)

= или == — строки равны

!= — строки не равны

< и > — лексикографическое сравнение (в [[ ]])

Числовые проверки:

-eq — равно (equal)

-ne — не равно (not equal)

-lt — меньше (less than)

-le — меньше или равно

-gt — больше (greater than)

-ge — больше или равно

Логические операторы:

&& — И (AND) — в [ ] использовать -a, в [[ ]] — &&

|| — ИЛИ (OR) — в [ ] использовать -o, в [[ ]] — ||

! — НЕ (NOT)

Рекомендация: Используйте двойные квадратные скобки [[ ]] — они мощнее и безопаснее.

Варианты заданий
Вариант 1 (Администратор файловой системы)
Напишите скрипт, который:

Принимает путь к файлу или директории как аргумент командной строки.

Проверяет, существует ли переданный объект.

Если объект существует:

Определяет его тип (файл или директория).

Проверяет права доступа (чтение, запись, выполнение).

Если это файл, проверяет, пустой ли он, и выводит его размер в байтах.

Если это директория, выводит количество элементов в ней (используя ls -1 | wc -l).

Если объект не существует — выводит сообщение об ошибке и завершается с кодом 1.

Все сообщения должны быть информативными и понятными пользователю.

Пример вывода:
```

$ ./script.sh /home/user/file.txt
Объект /home/user/file.txt существует.
Тип: обычный файл
Размер: 1024 байт
Пустой: нет
Права: чтение (+), запись (+), выполнение (-)

$ ./script.sh /nonexistent
Ошибка: /nonexistent не существует!

```

Вариант 2 (Калькулятор с проверками)
Напишите скрипт-калькулятор, который:

Принимает три аргумента: число1, оператор, число2 (например, 5 + 3).

Проверяет, переданы ли все три аргумента. Если нет — выводит инструкцию по использованию и завершается с кодом 1.

Проверяет, являются ли первый и третий аргументы числами (целыми или с плавающей точкой). Если нет — выводит ошибку.

Выполняет операцию в зависимости от оператора (+, -, *, /):

Для деления проверяет, не равен ли делитель нулю. Если равен — ошибка.

Для умножения используйте \* или * в кавычках.

Выводит результат в формате: Число1 Оператор Число2 = Результат.

Использует bc или awk для вычислений с плавающей точкой (по желанию).

Пример вывода:
```
text
$ ./calc.sh 10 + 5
10 + 5 = 15

$ ./calc.sh 10 / 3
10 / 3 = 3.33

$ ./calc.sh 10 / 0
Ошибка: деление на ноль!

$ ./calc.sh 10 a 5
Ошибка: аргумент 'a' не является числом!

```
Вариант 3 (Мониторинг системы)
Напишите скрипт мониторинга, который:

Принимает порог загрузки CPU (в процентах) как аргумент.

Если аргумент не передан, устанавливает порог по умолчанию = 80%.

Проверяет, что переданное значение — число от 0 до 100.

Получает текущую загрузку CPU (среднюю за последнюю минуту) с помощью top -bn1 | grep "Cpu(s)" или mpstat.

Сравнивает текущую загрузку с порогом:

Если загрузка меньше порога — выводит зелёное сообщение "Система работает стабильно".

Если загрузка между порогом и порогом+10% — жёлтое "Загрузка выше нормы".

Если загрузка превышает порог+10% — красное "КРИТИЧЕСКАЯ ЗАГРУЗКА!".

В любом случае выводит текущее значение загрузки.

Подсказка: Извлеките значение %id (idle) из top и вычислите загрузку как 100 - idle.

Пример вывода:
```
text
$ ./monitor.sh 75
Текущая загрузка CPU: 82.5%
ВНИМАНИЕ: Загрузка выше нормы!

$ ./monitor.sh
Текущая загрузка CPU: 45.3%
Система работает стабильно (порог: 80%)

```
Вариант 4 (Управление пользователями)
Напишите скрипт для управления пользователями, который:

Принимает команду (create, delete, check) и имя пользователя как аргументы.

Если аргументов меньше 2 — выводит справку.

Для команды check:

Проверяет, существует ли пользователь (команда id).

Если существует — выводит UID, GID, домашнюю директорию и оболочку.

Если не существует — сообщает об этом.

Для команды create:

Проверяет, существует ли пользователь.

Если существует — ошибка.

Если не существует — создает пользователя с домашней директорией и оболочкой /bin/bash (только для демонстрации, в реальности требует прав root).

Для команды delete:

Проверяет, существует ли пользователь.

Если не существует — ошибка.

Если существует — запрашивает подтверждение (используйте read -p) и удаляет пользователя.

Все проверки выполняются через if и id.

Пример вывода:
```
text
$ ./user_manager.sh check john
Пользователь john не найден.

$ ./user_manager.sh check root
Пользователь root найден:
  UID: 0
  GID: 0
```
  Домашняя директория: /root
  Оболочка: /bin/bash
